## Build Philosophy

1. **Each phase ships something usable.** No phase leaves the app broken. After any phase, you can demo what you've built.
2. **Infrastructure before features.** Phase 0 sets up everything so feature work is fast and clean.
3. **Magic moment first.** The Tell Me How demo (one sentence → grounded steps on a phone) works by the end of Phase 2. Everything else builds on top.
4. **Test as you go.** Each task's Notes end with a verification step. Don't save testing for the end.
5. **Progressive enhancement.** Simplest working version first, polish layered on in Phase 4.
6. **Review before proceeding.** Push each completed phase as a PR (`phase-{N}/{slug}`) and let a review agent (e.g. CodeRabbit) review it before starting the next phase.
7. **The one-sentence ceiling is law.** If a task adds required manager input beyond a card title, it's mis-specified — fix the task, not the principle (vision § Product Principles).
8. **Never write to `task_events` directly.** All event logging goes through `lib/events.ts` / the `log_task_event` RPC (PRD FR-005). The pilot gate depends on this log being complete and consistent.
9. **Ship the fall pilot.** Student workers return in late August. At the mid-August checkpoint, anything not blocking the magic moment or the nudge loop gets cut to Phase 4+ without debate.

## Agent Session Guide

### How to Use This Roadmap with Your Coding Agent

1. **Start a session:** Give your coding agent the phase prompt at the beginning of each phase.
2. **Read selectively:** Each phase lists its Reference sections — the specific parts of the PRD and vision doc needed for that phase. The agent should read only those sections, not the entire documents.
3. **Let it work:** The agent reads the roadmap, finds the first unchecked task, implements it, and marks it complete.
4. **One session = one phase (ideally):** If you need to stop, the agent resumes from the last unchecked task.
5. **Push a PR for review:** When a phase is complete, push the work as a PR (`phase-{N}/{slug}`) and let a review agent (e.g. [CodeRabbit](https://coderabbit.ai)) review it before starting the next phase. Address feedback on the same branch; merge only when clean.
6. **Need more context?** If a task references a section not in the phase's Reference sections, read just that section on demand.

### Session Tips

- **Don't read everything:** the Reference sections tell the agent exactly what to load; the full PRD wastes context.
- **Don't skip tasks:** ordering resolves dependencies. TASK-002 (Telnyx A2P) is first-week critical despite being a founder task.
- **Verify after each phase:** run the app and the E2E suite before moving on.
- **Update the status line** after completing tasks: `**Status:** X/62 tasks complete`, `**Current Phase:** Phase N`.
- **Mid-August checkpoint (hard rule):** if Phase 3 isn't complete by ~Aug 15, cut FR-019 (digest), FR-018 (landing page), and the P1/P2 edge-case sweep to after the pilot starts. The pilot needs Phases 0–3 plus TASK-044/045/046/056–059 only.
